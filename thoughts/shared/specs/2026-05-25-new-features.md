## P0 — Делать в первую очередь

### 1. AI-генерация вопросов для квиза

**Проблема:** Администратор вручную придумывает вопросы и вводит тайм-коды. Это самый трудоёмкий этап при создании курса.

**Решение:**
- В интерфейсе редактора урока — поле «Вставьте скрипт/описание видео»
- Кнопка «Предложить вопросы» → AI генерирует 3–5 вопросов с вариантами ответов и правильным ответом
- Администратор редактирует, удаляет лишнее, принимает нужное
- Тайм-коды всё ещё вводятся вручную (AI их не знает)

**Реализация:**
- Вызов Anthropic API (`claude-haiku` — дёшево и быстро) из серверной функции
- Промпт: «На основе этого текста придумай {N} вопросов с 4 вариантами ответа. Один правильный. Формат JSON.»
- Ответ парсится и подставляется в форму редактора

**Ценность:** Экономит 50–70% времени при создании курса.

---

### 2. Сброс прогресса при обновлении курса

**Проблема:** Если администратор меняет видео или вопросы — сотрудники, уже прошедшие курс, должны пройти его заново. Сейчас механизма нет.

**Решение:**
- При изменении урока (видео, вопросы, тайм-коды) — кнопка «Обновить курс и сбросить прогресс»
- Система сбрасывает `completedLessons`, `lessonProgress` и `quizAnswers` для всех, кто проходил этот курс
- Уведомление: «Курс обновлён. Прогресс {N} сотрудников сброшен.»

**Реализация:**
- Серверная функция `resetCourseProgressFn`
- `DELETE FROM lesson_progress WHERE lesson_id IN (...)`
- `DELETE FROM completed_lessons WHERE lesson_id IN (...)`
- Транзакция — всё или ничего

**Ценность:** Закрывает реальный кейс, который уже возникает в работе.

---

### 3. HR-отчёт о прохождении обучения

**Проблема:** Нет способа документально подтвердить, что конкретный сотрудник прошёл конкретный курс.

**Решение:**
- В разделе статистики — кнопка «Скачать отчёт»
- Выгружается Excel-файл со столбцами:
  - ФИО сотрудника
  - Курс
  - Дата прохождения
  - Результат финального теста (% правильных ответов)
  - Статус (Завершён / В процессе / Не начат)
- Фильтры: по курсу, по дате

**Реализация:**
- Библиотека `exceljs` (уже популярна в экосистеме Node)
- Серверная функция формирует файл и отдаёт base64 или blob
- На клиенте — скачивание через `<a href="blob:...">` или прямой ответ сервера

**Ценность:** HR-отдел получает документ для аудита и отчётности без ручной работы.

## P1 — Делать во вторую очередь

### 4. Массовое назначение курса («Назначить всем»)

**Проблема:** Чтобы назначить курс всем сотрудникам, нужно делать это по одному.

**Решение:**
- На странице курса — кнопка «Назначить всем сотрудникам»
- Подтверждение: «Будет создано {N} ссылок. Продолжить?»
- После подтверждения — массовая генерация персональных ссылок

**Реализация:**
- `INSERT INTO enrollments SELECT ... FROM employees WHERE companyId = ?` — один запрос
- Ссылки уже генерируются по токену — логика та же, просто в цикле на сервере

**Ценность:** Убирает рутину при старте нового курса для всей команды.

---

### 5. Дедлайн для курса

**Проблема:** Нет способа задать срок, к которому сотрудник должен завершить курс.

**Решение:**
- В настройках курса — поле «Дата завершения»
- В личном кабинете сотрудника — отображается «До {дата}»
- В статистике администратора — колонка «Просрочено» (красный маркер)

**Реализация:**
- Поле `deadline: timestamp` в таблице `enrollments` (или `courses`)
- На странице `/learn/$token` — отображение countdown или даты
- В статистике — фильтр «Просроченные»

**Ценность:** Контроль без лишних разговоров — администратор видит, кто не уложился.

---

### 6. Повторное прохождение по расписанию

**Проблема:** Часть курсов нужно повторять периодически (например, инструктаж по безопасности раз в год).

**Решение:**
- В настройках курса — «Повторять каждые: [6 месяцев / 1 год / вручную]»
- По истечении срока — прогресс сбрасывается автоматически, сотрудник видит курс снова

**Реализация:**
- Поле `repeatIntervalDays` в таблице `courses`
- Cron-задача (или scheduled function в Neon/Vercel) раз в сутки проверяет просроченные записи и сбрасывает прогресс

**Ценность:** Убирает ручную работу при повторяющихся обязательных инструктажах.

## P2 — Делать позже, если вырастет потребность

### 7. Импорт + назначение курса в одном шаге

При загрузке Excel с сотрудниками — сразу выбрать курс для назначения.
Новый сотрудник появляется в системе уже с назначенным курсом и ссылкой.

### 8. Прогресс-бар учебного пути

Если сотрудник проходит цепочку курсов — показывать общий прогресс по всему пути (Курс 1 ✓ → Курс 2 → Курс 3).

### 9. Краткое резюме курса для администратора

После того как все сотрудники завершили курс — автоматически формировать сводку: средний балл, среднее время прохождения, самый часто неверно отвечаемый вопрос.

### 10. Ограничение по количеству попыток финального теста

Сейчас неясно, сколько раз можно пересдавать финальный тест. Опционально — ограничить (например, 3 попытки), после чего — блокировка и уведомление администратора.

## Рекомендуемый порядок внедрения

```
Неделя 1:  HR-отчёт (Excel) — минимальная реализация
Неделя 2:  Сброс прогресса при обновлении курса
Неделя 3:  AI-генерация вопросов
Неделя 4:  Массовое назначение + дедлайн
```

Начинаем с HR-отчёта, потому что это закрытый, изолированный модуль с высокой ценностью для клиента — легко показать результат.
